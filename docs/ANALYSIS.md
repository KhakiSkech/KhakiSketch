# KhakiSketch 프로젝트 종합 분석 및 개선 계획

## 📊 현재 상태 분석

### 1. 블로그 대시보드 0 표시 문제 🔴

**현상**: 대시보드에서 블로그 글 수가 0으로 표시됨 (실제로 3개 존재)

**원인 분석**:
```typescript
// app/admin/page.tsx - 대시보드
const articlesRes = await getAllArticles(); // Firestore 조회
setStats({ articles: articlesRes.length }); // 0 표시

// lib/firestore-articles.ts - getAllArticles()
// Firebase 실패 시 MOCK_ARTICLES 반환 (4개)
// 성공하지만 빈 배열 반환 시 0개로 표시
```

**문제점**:
1. **데이터 불일치**: 
   - `MOCK_ARTICLES` (lib/mock-articles.ts): 4개
   - `staticArticles` (data/articles.ts): 3개
   - Firestore: 0개 (실제로 저장된 데이터 없음)

2. **Fallback 로직 문제**:
   - Firebase 연결 실패 시에만 Mock 데이터 사용
   - 연결 성공했지만 데이터가 없으면 빈 배열 반환

**해결책**:
- ✅ Firestore에 실제 데이터 마이그레이션
- ✅ Fallback 로직 개선 (정적 데이터 우선 사용 또는 병합)
- ✅ 대시보드에 데이터 소스 표시 (Firestore/정적)

### 3. Quotes CRM 개선 방안 💡

#### 현재 기능
- ✅ 리드 목록 및 필터링 (상태별)
- ✅ 상세 페이지 - 고객/프로젝트 정보
- ✅ 상태 파이프라인 관리 (NEW → WON/LOST)
- ✅ 우선순위 설정 (LOW ~ URGENT)
- ✅ 타임라인/활동 로그
- ✅ 노트 시스템
- ✅ 기본 통계 (총 견적, 이번 달, 신규, 계약완료)

#### 제안하는 개선사항

**A. 견적서 발송 기능** ⭐⭐⭐
```
- 이메일 템플릿 기반 견적서 작성
- PDF 생성 및 첨부
- 발송 이력 자동 기록
- 견적서 상태 추적 (발송 → 확인 → 승인)
```

**B. 할 일/리마인더 시스템** ⭐⭐⭐
```
- 리드별 할 일 등록
- 기한 설정 및 알림
- 대시보드에 오늘의 할 일 표시
- 이메일/브라우저 알림
```

**C. 칸반 보드 뷰** ⭐⭐
```
- 상태별 드래그 앤 드롭
- 파이프라인 시각화
- 직관적인 진행 상황 파악
```

**D. 고객 이력 관리** ⭐⭐
```
- 이전 견적/프로젝트 연결
- 고객별 통계 (총 계약액, 성공률)
- 반복 고객 식별
```

**E. 팀 할당** ⭐⭐
```
- 리드 담당자 지정
- 팀원별 성과 통계
- 권한 관리 (자신의 리드만 보기)
```

**F. 고급 필터링/검색** ⭐
```
- 예산 범위 필터
- 프로젝트 유형 필터
- 키워드 검색
- 날짜 범위 필터
```
