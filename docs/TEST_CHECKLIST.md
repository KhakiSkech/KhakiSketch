## 테스트 시나리오

### 1. Firebase 연결 테스트
**목적**: Firebase 서비스 정상 연결 확인

#### 1.1 앱 초기화 테스트
```bash
# 테스트 방법
npm run dev
```

**예상 결과**:
- [ ] 브라우저 콘솔에 Firebase 초기화 에러 없음
- [ ] `Firebase: Error (auth/invalid-api-key)` 없음
- [ ] 앱 정상 로드됨

#### 1.2 Firestore 연결 테스트
**테스트 단계**:
1. 메인 페이지 접속 (`/`)
2. 브라우저 개발자 도구 > Console 확인
3. Firestore 관련 에러 메시지 확인

**통과 기준**:
- [ ] `FirebaseError: Missing or insufficient permissions` (보안 규칙 설정 전 예상됨)
- [ ] Connection 에러 없음

**상태**: ⏳ 대기중 (실제 Firebase 설정 필요)

### 2. 관리자 로그인 흐름 테스트
**목적**: Firebase Auth 기반 관리자 인증 확인

#### 2.1 로그인 페이지 접근
**테스트 단계**:
1. `/admin` 접속
2. 로그인 폼 표시 확인

**통과 기준**:
- [ ] 이메일/비밀번호 입력 필드 표시
- [ ] "관리자 로그인" 제목 표시
- [ ] KhakiSketch 로고 표시

#### 2.2 로그인 성공 케이스
**테스트 단계**:
1. 등록된 관리자 이메일 입력
2. 올바른 비밀번호 입력
3. 로그인 버튼 클릭

**예상 결과**:
- [ ] 로딩 상태 표시 ("로그인 중...")
- [ ] 대시보드로 리다이렉트
- [ ] 상단 네비게이션에 "로그아웃" 버튼 표시
- [ ] 사용자 이름/이메일 표시

#### 2.3 로그인 실패 케이스
**테스트 단계**:
1. 잘못된 이메일 입력
2. 잘못된 비밀번호 입력

**예상 결과**:
- [ ] 에러 메시지 표시 ("이메일 또는 비밀번호가 올바르지 않습니다")
- [ ] 폼 초기화되지 않음
- [ ] 로그인 버튼 재활성화

#### 2.4 로그아웃 테스트
**테스트 단계**:
1. 관리자 로그인
2. "로그아웃" 버튼 클릭

**예상 결과**:
- [ ] 로그인 페이지로 리다이렉트
- [ ] 세션/토큰 삭제 확인

**상태**: ⏳ 대기중 (실제 Firebase Auth 설정 필요)

### 3. 견적 제출 및 저장 테스트
**목적**: 견적 Wizard에서 Firestore까지 데이터 흐름 확인

#### 3.1 견적 Wizard 완료
**테스트 단계**:
1. `/quote` 접속
2. Step 1: 프로젝트 유형 선택 (예: "MVP 개발")
3. Step 2: 프로젝트 상세 정보 입력
4. Step 3: 기술 스택/플랫폼 선택
5. Step 4: 예산/일정 선택 + 연락처 입력
6. "견적 요청하기" 버튼 클릭

**예상 결과**:
- [ ] 로딩 상태 표시 ("전송 중...")
- [ ] 성공 화면 표시 ("견적 요청이 접수되었습니다!")
- [ ] 요약 정보 표시

#### 3.2 Firestore 저장 확인
**테스트 단계**:
1. Firebase Console > Firestore Database 접속
2. `quote-leads` 컬렉션 확인

**통과 기준**:
- [ ] 새 문서 생성됨
- [ ] 모든 필드 정상 저장:
  - [ ] customerName, email, phone
  - [ ] projectType, projectName
  - [ ] budget, timeline
  - [ ] status: "NEW"
  - [ ] createdAt: 타임스탬프

#### 3.3 관리자 대시보드 표시
**테스트 단계**:
1. 관리자 로그인
2. `/admin/quotes` 접속

**통과 기준**:
- [ ] 새 견적 목록에 표시
- [ ] 상태: "접수" (NEW)
- [ ] 고객 이름, 프로젝트 유형 표시

#### 3.4 견적 상세 페이지
**테스트 단계**:
1. 견적 목록에서 상세 보기 클릭
2. `/admin/quotes/{id}` 접속

**통과 기준**:
- [ ] 고객 정보 표시
- [ ] 프로젝트 정보 표시
- [ ] 기술 요구사항 표시
- [ ] 상태 변경 버튼 작동
- [ ] 노트 추가 기능 작동

**상태**: ⏳ 대기중 (실제 Firestore 연결 필요)

## 🔧 자동화 테스트 (권장)

### Playwright 설치 및 설정
```bash
npm install --save-dev @playwright/test
npx playwright install
```

### 기본 테스트 예시
```typescript
// tests/admin-login.spec.ts
import { test, expect } from '@playwright/test';

test('관리자 로그인 성공', async ({ page }) => {
  await page.goto('http://localhost:3000/admin');
  await page.fill('input[type="email"]', 'admin@khakisketch.com');
  await page.fill('input[type="password"]', 'password123');
  await page.click('button:has-text("로그인")');
  await expect(page).toHaveURL(/.*admin/);
  await expect(page.locator('text=대시보드')).toBeVisible();
});
```
